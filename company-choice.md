Company Choice

Which company I picked: Nexova Solutions

I chose Nexova Solutions. Nexova is a human resources and talent company that helps other
businesses hire people, train their staff, and run customer support teams. I picked it because
almost every department has a clear, real AI problem to solve — reading and scoring CVs,
searching through candidate profiles, building a support chatbot, and building recommendation
systems. This gives me a chance to work on many different AI skills, like search, chatbots,
scoring, and dashboards, all inside one solid project. I also liked that the problems are clearly
defined and connected to real daily work, so I can see exactly how AI would make a difference for
the people using it.

Two departments I think are the most interesting


Talent Selection (hiring team - Javier Almeida) — Consultants currently read CVs one by
one by hand, which takes a lot of time. I want to build a system that scores and ranks CVs
automatically, and lets you search for candidates using natural language, like "find someone
with sales experience and English C1."
Customer Support (Roberto Díaz) — This team answers customer questions all day, and many
of them are repeated questions. Building a chatbot that can answer the common questions on its
own would improve response time and help the team meet their service deadlines.


The AI challenge I'm most excited to build

The CV scoring and searching system for the hiring team. Right now, consultants read 30 to 80
CVs by hand for every job opening. I want to build a system where you type what you're looking
for, like "3 years of sales experience and English C1," and it finds and ranks the best matching
candidates. This project combines search, AI, and real data, which makes it a strong system to
build and demonstrate.

My idea for an AI Agent

I want to build a Candidate Matching Agent. Recruiters would type a request into a chat, like
"find someone with sales experience who speaks English well," and the agent would search through
the candidate database and return a ranked list of the best matches. To do this, it would need
access to the CV database and the list of open job vacancies, so it can compare candidates against
the right requirements. It would run inside the recruiter's daily workflow, triggered whenever a
new job opens, and it would explain the reasoning behind each match so the recruiter can make the
final decision with confidence.
